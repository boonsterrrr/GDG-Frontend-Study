# 서버와 통신하기 - API가 뭐지. 프론트/백엔드는 머지

JsonPlaceholder라는 사이트 참고하기

## 용어 정리

클라이언트 ---요청(request): 서버야 ~~데이터 보내줭 ---> 서버
<--- 응답(response): 클라이언트야 데이터 보냈엉.

-> 클라이언트와 서버는 데이터를 주고 받는 서로의 상대적인 역할 놀이.

### API(Application Programming Interface)

: 클라이언트와 서버 간의 데이터를 주고 받는 통로 역할.
: 서로 다른 소프트웨어인 두 역할간 상호작용을 돕는 인터페이스.

### 프론트엔드

: Front + End (맨 앞)
: 사용자가 직접 상호작용하는 웹 페이지의 화면을 다루는 개발 영역

### 백엔드

: Back + End (맨 뒤)
: 서버에서 동작하는 부분을 다루는 개발 영역
: 비즈니스 로직 처리,인증, 보안 등을 다룸.
